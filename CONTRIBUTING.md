# Contributing

Contributions are welcome if they keep the pack small, Codex-native, and useful
on real software work.

## Good Contributions

- Improve an existing skill without making it much longer.
- Add concrete examples that make a workflow easier to follow.
- Remove Claude-specific or tool-specific assumptions.
- Add a new skill only when it covers a repeatable engineering workflow.

## Bar For New Skills

- One clear job.
- Strong `description` with trigger language.
- No hidden external dependencies.
- No broad process framework.
- No fake automation, spam, or engagement manipulation.

## Before Opening A PR

- Read the skill aloud once and remove filler.
- Keep `SKILL.md` focused on the core workflow.
- Put optional UI metadata in `agents/openai.yaml`.
- Test manual installation into a temporary `CODEX_HOME`.
