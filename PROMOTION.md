# Promotion Kit

Use this file for ethical launch and SEO work. The goal is to make the project
easy to discover, evaluate, install, and share. Do not use fake stars, bot
accounts, paid engagement, comment spam, or irrelevant backlink drops.

## 12 Promotion Methods

1. GitHub repository metadata: keep the description and topics focused on
   `codex`, `codex-skills`, `openai-codex`, `skill-md`, `ai-agents`,
   `software-engineering`, `code-review`, `security`, and `performance`.
2. SEO landing page: publish the GitHub Pages site at
   `https://arcqian-bit.github.io/codex-real-engineer-skills/`.
3. Long-tail install page: target searches such as "install Codex skill" and
   "coolest Codex apps", "install Codex skill", and "Codex skill collection".
4. Comparison page: target people searching for Codex alternatives to
   Claude-oriented skill workflows.
5. Examples page: show concrete before/after prompts and expected behavior.
6. Sitemap and robots: expose crawlable URLs under `docs/sitemap.xml` and
   `docs/robots.txt`.
7. Structured data: include JSON-LD on the landing page so search engines can
   understand the project as open-source software.
8. LLM discovery: publish `llms.txt` so AI agents can summarize and cite the
   project accurately.
9. GitHub release: publish a first release with clear install instructions and
   a concise changelog.
10. Community posts: share in communities where Codex users already ask about
    workflows, testing, debugging, and AI coding agents.
11. Directory submissions: submit to awesome lists, Codex resource indexes, AI
    coding tool directories, newsletters, and weekly issue roundups.
12. Feedback loop: invite issues with real examples, then turn good feedback
    into README examples, FAQ entries, and release notes.
