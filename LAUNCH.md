# Launch Notes

Legitimate ways to grow stars:

1. Rename the repository to `codex-real-engineer-skills`.
2. Set the repository description to:
   `One Codex skill for real engineering: diagnose, TDD, grill, prototype, and architecture review.`
3. Add topics:
   `codex`, `codex-skills`, `openai-codex`, `skill-md`, `ai-agents`,
   `software-engineering`, `tdd`, `debugging`, `architecture`.
4. Share the repo in communities where Codex users already discuss skills:
   `r/codex`, `r/OpenAICodex`, `r/ChatGPTCoding`, Hacker News Show HN,
   LinkedIn, X, and relevant Discord/Slack groups.
5. Submit it to Codex/agent skill directories and awesome lists.
6. Ask for feedback before asking for stars.

Short launch post:

```text
I made a Codex-native adaptation of Matt Pocock-style "real engineering"
workflows.

It is one `real-engineer` SKILL.md with five modes:
- diagnose: reproduce before guessing
- tdd: red/green/refactor vertical slices
- grill: clarify fuzzy requirements
- prototype: disposable UI/logic prototypes
- architecture: find deeper modules and better seams

No Claude-specific setup, no duplicate skill triggers, no big framework.

https://github.com/ArcQian-bit/test
```

Polite outreach template:

```text
Hey, I saw you were collecting Codex / SKILL.md resources. I built a small
Codex-native `real-engineer` skill adapted from "real engineering" workflows:

https://github.com/ArcQian-bit/test

Would this fit your list? Feedback welcome either way.
```

Do not use fake stars, paid stars, bot accounts, or drive-by spam. Those signals
are easy to spot and make the project less trustworthy.
