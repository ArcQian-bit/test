# Launch Notes

Legitimate ways to grow stars:

1. Present the project as `Coolest Codex Apps`.
2. Set the repository description to:
   `10 Codex SKILL.md apps for repo mapping, issue triage, PR review, security, performance, releases, docs, UI, migrations, and real engineering.`
3. Add topics:
   `codex`, `codex-skills`, `openai-codex`, `skill-md`, `ai-agents`,
   `software-engineering`, `code-review`, `security`, `performance`, and
   `migration`.
4. Share the repo in communities where Codex users already discuss skills:
   `r/codex`, `r/OpenAICodex`, `r/ChatGPTCoding`, Hacker News Show HN,
   LinkedIn, X, and relevant Discord/Slack groups.
5. Submit it to Codex/agent skill directories and awesome lists.
6. Ask for feedback before asking for stars.

Short launch post:

```text
I upgraded my Codex skill repo into Coolest Codex Apps: 10 focused SKILL.md apps
for real repo work.

It includes repo mapping, issue triage, PR review, security review, performance
work, release prep, docs sync, UI polish, migration planning, and the original
real-engineer fallback workflow.

No Claude-specific setup, no slash commands, no overlapping trigger soup.

https://github.com/ArcQian-bit/codex-real-engineer-skills
```

Polite outreach template:

```text
Hey, I saw you were collecting Codex / SKILL.md resources. I built Coolest Codex
Apps, a small Codex-native collection of 10 engineering workflow skills:

https://github.com/ArcQian-bit/codex-real-engineer-skills

Would this fit your list? Feedback welcome either way.
```

Do not use fake stars, paid stars, bot accounts, or drive-by spam. Those signals
are easy to spot and make the project less trustworthy.
